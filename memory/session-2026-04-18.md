# Session Notes — 2026-04-18

## Features added this session

### 1. Exclude `from-references` papers from stats and recent
- `routers/stats.py` — all four queries (paper count, by_year, top_topics, recent_papers)
  now filter out papers tagged `from-references`.

### 2. Work Anthropic account (Palantir/Merck gateway) fixed
- Root cause: `chat_with_paper_work()` was not passing `verify=False` to httpx,
  so Zscaler intercepted the TLS handshake and returned 401.
- Fix: added `_ssl_verify()` helper and `http_client=httpx.Client(verify=_ssl_verify())`
  to `chat_with_paper_work()` in `services/ai.py`.
- Confirmed working: `claude-opus-4-6` is the only model available on the Palantir gateway
  (claude-3-5-sonnet and others return 404).
- Requires backend restart to pick up code changes.

### 3. Markdown rendering for AI summary and chat
- Installed `@tailwindcss/typography` and added `@plugin "@tailwindcss/typography"` to
  `frontend/src/index.css` (Tailwind v4 syntax) — activates `prose` classes.
- `PaperDetail.tsx` summary section: replaced plain `<p>` with `<ReactMarkdown>` inside
  a `prose prose-sm` div.
- `ChatPanel.tsx` already had ReactMarkdown — prose classes now actually render.

### 4. Externalised AI prompts to `prompts/` folder
- Created `prompts/` at project root (next to `backend/`):
  - `prompts/summary.txt` — uses `{title}` and `{text}` placeholders
  - `prompts/chat_system.txt` — uses `{title}` and `{text}` placeholders
  - `prompts/topics.txt` — uses `{title}` and `{context}` placeholders
- `services/ai.py` loads prompts via `_load_prompt(filename)` on every call
  (no restart needed to pick up edits).

### 5. Automatic AI topic suggestion
- `services/ai.py` — new `suggest_topics(title, abstract, summary)` using Claude Haiku.
  Regex extracts JSON even if Claude wraps it in markdown fences.
- `routers/topics.py` — new `POST /papers/{id}/topics/suggest` endpoint.
- Both upload pipelines (`/papers/upload` and `/papers/from-url`) now auto-suggest topics
  after ingestion (best-effort, non-blocking, deduped against S2 topics).
- `api/client.ts` — new `suggestTopics(paperId)` helper.
- `PaperDetail.tsx` — "✦ Suggest topics with AI" button in Topics section with
  chip-select UI and "Apply N topics" confirm button. Already-linked topics shown as ✓.

### 6. Library Maintenance / Backfill pattern
- `routers/backfill.py` — `POST /backfill/topics` and `POST /backfill/summary`.
  Each skips papers that already have the data; returns `{processed, skipped, errors}`.
- `Settings.tsx` — new "Library Maintenance" section with a `BackfillRow` component
  per operation showing live run status and result counts.
- **Rule saved to memory**: when adding any AI enrichment feature, always ask whether
  to apply it to existing papers, and add a backfill endpoint + Settings row if yes.
- First run backfilled topics for 77 existing papers.

## Key decisions / gotchas
- Backfill router uses prefix `/backfill` (not `/papers/backfill`) to avoid route
  conflict with the papers router's `/{paper_id}/...` wildcard.
- Tailwind v4 uses `@plugin` directive in CSS (not `plugins: []` in config file).
- `suggest_topics` uses Claude Haiku (fast/cheap), not Opus.
- Prompts are loaded fresh on every request — edit files, see changes immediately.
